Imatges Docker
Construcció i Gestió

Passa de ser "usuari" de contenidors a "creador". Aprèn el sistema de capes, Docker Hub i com crear les teves pròpies imatges amb un Dockerfile.

Conceptes clau

📦 Què és una imatge?

Si un contenidor és l'execució (la instància), la imatge és la plantilla immutable.

🧅 Sistema de Capes

Les imatges es construeixen com una ceba: cada instrucció del Dockerfile afegeix una capa nova. Això les fa molt eficients per emmagatzemar i distribuir.

📦 Tot inclòs

Conté el sistema operatiu base, llibreries, codi i configuracions. Si funciona a la teva màquina, funcionarà a qualsevol lloc amb Docker instal·lat.

🌐 Docker Hub: El repositori global

Docker Hub és el registre públic on trobem imatges oficials (Nginx, Python, Ubuntu, Node.js, MySQL, etc.).

# Descarregar una imatge (sense executar)
docker pull ubuntu:22.04

# Executar de forma interactiva
docker run -it ubuntu:22.04 bash

# Llistar imatges locals
docker images

# Buscar imatges al Hub
docker search nginx
Crear imatges pròpies

📝 Sintaxi del Dockerfile

InstruccióDescripcióExemple
FROMImatge base (obligatòria, primera línia)FROM ubuntu:24.04
RUNExecuta comandes durant la construccióRUN apt-get update && apt-get install -y python3
COPYCopia fitxers del host a la imatgeCOPY script.py /app/
ADDCom COPY però pot descomprimir i descarregar URLsADD fitxer.tar.gz /app/
WORKDIREstableix el directori de treballWORKDIR /app
CMDComanda per defecte (sobreescribible)CMD ["python3", "app.py"]
ENTRYPOINTComanda principal (no sobreescribible)ENTRYPOINT ["python3"]
EXPOSEDocumenta el port que usa l'aplicacióEXPOSE 80
ENVVariable d'entorn persistentENV PORT=3000

🛠️ Taller pràctic: Primera imatge personalitzada

Crearem una imatge basada en Ubuntu 24.04 que executa un script Python automàticament.

Pas A: Preparar els fitxers

Fitxer script.py:

print("Hello, World! Sóc un contenidor personalitzat.")

Fitxer Dockerfile (sense extensió):

# 1. Imatge base
FROM ubuntu:24.04

# 2. Instal·lem dependències
RUN apt-get update && apt-get install -y python3

# 3. Copiem el codi dins la imatge
COPY script.py /app/

# 4. Comanda que s'executa en arrencar
CMD ["python3", "/app/script.py"]

Pas B: Construir la imatge

# El punt (.) indica que el Dockerfile és al directori actual
docker build -t la-meva-imatge:v1 .

# Verificar que s'ha creat
docker images

# Executar-la
docker run --rm la-meva-imatge:v1
