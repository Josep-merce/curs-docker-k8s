Ingress
Enrutament Avançat

A internet no anem teclejant IPs amb ports estranys. Amb Ingress i Traefik, accedim a les aplicacions amb dominis nets com app.local.

Limitacions del NodePort

🚧 Per què necessitem Ingress?

  • Ports poc amigables: Obliga a ports alts (30000+). Els usuaris esperen el port 80/443.
  • Gestió caòtica: 15 microserveis = 15 ports diferents a recordar.
  • Manca d'enrutament: NodePort no entén de dominis ni rutes (/api, /blog).
Ingress + Traefik

🚦 Com funciona?

📋 L'Ingress (Les regles)

Fitxer YAML on definim les normes: "Si demanen app.local, envia-ho a Nginx. Si demanen api.local, envia-ho al backend."

⚙️ Ingress Controller (El motor)

El programa real (reverse proxy) que llegeix les regles i executa el trànsit. k3s ve amb Traefik preinstal·lat.

Pràctica: El teu primer domini local

🛠️ Configurar app.local

0

Preparar el Deployment

apiVersion: apps/v1
kind: Deployment
metadata:
  name: el-meu-web-deployment
spec:
  replicas: 1
  selector:
    matchLabels:
      app: el-meu-web
  template:
    metadata:
      labels:
        app: el-meu-web
    spec:
      containers:
      - name: nginx
        image: nginx:latest
        ports:
        - containerPort: 80
1

Service intern (ClusterIP)

Crea nginx-clusterip.yaml:

apiVersion: v1
kind: Service
metadata:
  name: servei-web-intern
spec:
  type: ClusterIP
  selector:
    app: el-meu-web
  ports:
  - port: 80
    targetPort: 80
kubectl apply -f nginx-clusterip.yaml
2

Crear la regla d'Ingress

Crea nginx-ingress.yaml:

apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: ingress-web
  annotations:
    traefik.ingress.kubernetes.io/router.entrypoints: web
spec:
  ingressClassName: traefik
  rules:
  - host: app.local
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: servei-web-intern
            port:
              number: 80
kubectl apply -f nginx-ingress.yaml
3

Verificar l'Ingress

kubectl get ingress
4

Obtenir la IP del node

kubectl get nodes -o wide
# Copia la columna INTERNAL-IP

Editar el fitxer hosts

Linux/macOS: sudo nano /etc/hosts
Windows: C:\Windows\System32\drivers\etc\hosts

<INTERNAL-IP> app.local
5

La prova final

Obre el navegador i accedeix a http://app.local 🎉

🎯 Exposa botiga.local

Crea un Deployment (httpd:alpine, 2 rèpliques), un Service ClusterIP i un Ingress perquè botiga.local apunti a la teva botiga. No oblidis editar el fitxer /etc/hosts!
