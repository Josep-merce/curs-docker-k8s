Introducció a Kubernetes
K8s i Pods

Kubernetes és el director d'orquestra dels contenidors. Aprèn l'arquitectura bàsica, instal·la k3s i llança el teu primer Pod.

Kubernetes vs k3s

☸️ Per què k3s per al curs?

CaracterísticaKubernetes (K8s clàssic)k3s (El nostre entorn)
Pes i RecursosAlt (molta memòria i CPU)Molt lleuger (<100MB de binari)
Instal·lacióComplexa, múltiples componentsSimple, un sol fitxer executable
Cas d'ús idealGrans centres de dades i produccióAprenentatge, Edge computing, proves locals
Arquitectura bàsica

🧠 Control Plane (Cervell)

El node mestre. Pren les decisions globals: on col·locar una nova aplicació, com reaccionar si una màquina s'espatlla.

  • Gestiona l'API (porta d'entrada per a les peticions)
  • Emmagatzema l'estat global del sistema

💪 Worker Nodes (Músculs)

Les màquines que realment executen les aplicacions (contenidors).

  • Reben instruccions del Control Plane
  • Informen constantment del seu estat
Posada en marxa amb Vagrant + k3s

💻 Arrencar el clúster

1

Clonar el repositori

git clone https://git.agile611.com/Agile611/startusingkubernetes.git
cd startusingkubernetes
2

Aixecar la infraestructura

vagrant up
# Pot trigar uns minuts la primera vegada. Paciència!
3

Accedir a la màquina principal

vagrant ssh server
kubectl — El comandament a distància

🎮 Primeres comandes

# Comprovar els nodes del clúster
sudo kubectl get nodes

# Informació del clúster
sudo kubectl cluster-info

# Llistar tots els Pods
sudo kubectl get pods

# Llistar tots els Pods de tots els namespaces
sudo kubectl get pods -A
Tasca pràctica

🏋️ Desplegament amb K3s

Crea un fitxer activitat9.yaml amb un Pod Apache i desplega'l:

apiVersion: v1
kind: Pod
metadata:
  name: pod-apache-k3s
spec:
  containers:
  - name: apache
    image: httpd:alpine
    ports:
    - containerPort: 80
sudo kubectl apply -f activitat9.yaml
sudo kubectl get pods
