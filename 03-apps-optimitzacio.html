Optimització: El concepte de "Runtime"

⚡ Imatge Base Correcta

No facis servir sempre ubuntu com a base. És pesat i insegur per a producció.

❌ Manera incorrecta (Pesada)

FROM ubuntu:24.04
RUN apt-get update && apt-get install -y python3
COPY script.py /app/
CMD ["python3", "/app/script.py"]

~800 MB

✅ Manera correcta (Runtime slim)

FROM python:3.12-slim
WORKDIR /app
COPY script.py .
CMD ["python", "script.py"]

~150 MB · 5× més petita

✅ Avantatges de les imatges slim/alpine
  • Mida: Passem de ~800MB a ~150MB (o ~50MB amb alpine)
  • Seguretat: Menys paquets innecessaris = menys vulnerabilitats
  • Velocitat: Es descarrega i construeix més ràpid
CMD vs ENTRYPOINT

🧠 Diferència clau

InstruccióComportamentEs pot sobreescriure?
CMDDefineix la comanda per defecteSÍ (fàcilment amb docker run ... nova-comanda)
ENTRYPOINTDefineix la comanda principal/fixaNO (els arguments s'afegeixen al final)
 
# Amb CMD ["echo", "Hola"]
docker run la-imatge Adéu   # Imprimeix: "Adéu" (substitueix)

# Amb ENTRYPOINT ["echo"]
docker run la-imatge Hola   # Imprimeix: "Hola" (concatena)
Eina: Dive

🔍 Anàlisi de capes amb Dive

Dive ens permet inspeccionar les capes d'una imatge i detectar espai malgastat.

# Instal·lació (Mac)
brew install dive

# Analitzar una imatge
dive node:alpine
dive python:3.12

Què buscar a Dive?

  • Efficiency Score: Com més alt, millor.
  • Wasted Space: Fitxers esborrats en capes posteriors però que ocupen espai a l'historial.
  • Layer Details: Quina comanda ha afegit més megabytes?
Tasca pràctica

🛠️ El Repte de l'Optimització

Actua com a enginyer DevOps: optimitza una imatge "fat" a "slim".

Part 1: La imatge "fat"

Crea app.py:

import sys
print(f"Hola! Estic executant-me amb els arguments: {sys.argv[1:]}")

Crea Dockerfile.fat:

FROM ubuntu:24.04
RUN apt-get update && apt-get install -y python3
WORKDIR /app
COPY app.py .
CMD ["python3", "app.py", "versio-fat"]
docker build -f Dockerfile.fat -t repte-python:fat .

Part 2: La imatge "slim"

Crea Dockerfile.slim amb imatge base oficial de Python, ENTRYPOINT en lloc de CMD:

FROM python:3.12-slim
WORKDIR /app
COPY app.py .
ENTRYPOINT ["python", "app.py"]
docker build -f Dockerfile.slim -t repte-python:slim .

Part 3: Comparació

# Comparar mides
docker images | grep repte-python

# Provar ENTRYPOINT amb argument
docker run --rm repte-python:slim Sóc-Un-DevOps-Expert

# Analitzar amb Dive
dive repte-python:fat
