K8s i Pods

Kubernetes és el director d'orquestra dels contenidors. Aprèn l'arquitectura bàsica, instal·la k3s i llança el teu primer Pod.

Arquitectura bàsica

🧠 Control Plane (Cervell)

El node mestre. Pren les decisions globals: on col·locar una nova aplicació, com reaccionar si una màquina s'espatlla.

  • Gestiona l'API (porta d'entrada per a les peticions)
  • Emmagatzema l'estat global del sistema

💪 Worker Nodes (Músculs)

Les màquines que realment executen les aplicacions (contenidors).

  • Reben instruccions del Control Plane
  • Informen constantment del seu estat
Instal·lació de k3s

💻 Instal·lar k3s directament

1

Instal·lar k3s (una sola comanda)

curl -sfL https://get.k3s.io | sh -

Espera 1-2 minuts. k3s instal·la i arrenca automàticament com a servei del sistema.

2

Configurar permisos de kubectl

Per no haver d'usar sudo cada vegada:

mkdir -p ~/.kube
sudo cp /etc/rancher/k3s/k3s.yaml ~/.kube/config
sudo chown $(id -u):$(id -g) ~/.kube/config
export KUBECONFIG=~/.kube/config

Per fer-ho permanent, afegeix l'última línia al teu ~/.bashrc o ~/.zshrc:

echo 'export KUBECONFIG=~/.kube/config' >> ~/.bashrc
source ~/.bashrc
3

Verificar que funciona

kubectl get nodes
# Ha de mostrar el node amb estat Ready
ℹ️ Gestió del servei k3s

k3s s'executa com a servei del sistema. Pots gestionar-lo amb:
sudo systemctl status k3s — veure l'estat
sudo systemctl stop k3s — aturar
sudo systemctl start k3s — arrencar

kubectl — El comandament a distància

🎮 Primeres comandes

# Comprovar els nodes del clúster
kubectl get nodes

# Informació del clúster
kubectl cluster-info

# Llistar tots els Pods
kubectl get pods

# Llistar tots els Pods de tots els namespaces
kubectl get pods -A
Tasca pràctica

🏋️ Desplegament amb K3s

Crea un fitxer activitat9.yaml amb un Pod Apache i desplega'l:

apiVersion: v1
kind: Pod
metadata:
  name: pod-apache-k3s
spec:
  containers:
  - name: apache
    image: httpd:alpine
    ports:
    - containerPort: 80
kubectl apply -f activitat9.yaml
kubectl get pods
