Docker Swarm
Orquestració i Clústers

Deixa de pensar en contenidors individuals i comença a pensar en Serveis. Forma un clúster que s'autorepara i escala automàticament.

Arquitectura del Swarm

🐝 Conceptes fonamentals

🧠 Manager (El Cervell)

Pren decisions, programa contenidors i manté l'estat del clúster.

💪 Worker (El Múscul)

Només executa les tasques que li mana el Manager.

Diferència clau

Docker normal: docker run (llança un contenidor). Docker Swarm: docker service create (defineix un estat desitjat: "vull 3 còpies sempre actives").

Crear el clúster

🛠️ Init & Join

1

Inicialitzar el Swarm

docker swarm init --advertise-addr <LA_TEVA_IP_PRIVADA>

Docker respondrà amb una comanda docker swarm join --token... Copia-la!

2

Afegir Workers (en altres màquines)

# A la màquina worker, enganxa la comanda del pas 1:
docker swarm join --token SWMTKN-xxx... IP:2377
3

Verificar l'estat

docker node ls

Hauries de veure el node amb estatus Leader.

Gestió de Serveis

🚀 Comandes de serveis

# Crear un servei amb 3 rèpliques
docker service create --name la-meva-web --replicas 3 -p 80:80 nginx

# Escalar (pujar a 5 rèpliques)
docker service scale la-meva-web=5

# Rolling Update (actualitzar sense parar el servei)
docker service update --image nginx:alpine la-meva-web

# Manteniment d'un node (treure'l del servei)
docker node update --availability drain <ID_DEL_NODE>

# Llistar serveis
docker service ls

# Veure on s'executa cada tasca
docker service ps la-meva-web
Projecte final: WordPress en Swarm

🏆 Stack WordPress amb docker stack deploy

version: "3.8"
services:
  db:
    image: mariadb:11
    environment:
      MYSQL_ROOT_PASSWORD: root_pass
      MYSQL_DATABASE: wordpress
      MYSQL_USER: wpuser
      MYSQL_PASSWORD: wp_pass
    volumes:
      - db_data:/var/lib/mysql
    deploy:
      placement:
        constraints:
          - node.labels.role == worker

  wordpress:
    image: wordpress:latest
    depends_on: [db]
    environment:
      WORDPRESS_DB_HOST: db:3306
      WORDPRESS_DB_USER: wpuser
      WORDPRESS_DB_PASSWORD: wp_pass
      WORDPRESS_DB_NAME: wordpress
    volumes:
      - wordpress_data:/var/www/html
    ports:
      - "80:80"
    deploy:
      placement:
        constraints:
          - node.labels.role == worker

volumes:
  db_data:
  wordpress_data:
# Etiquetar el node
docker node update --label-add role=worker <HOSTNAME>

# Desplegar el stack
docker stack deploy -c docker-compose.yml el-meu-blog

# Verificar
docker service ls
docker stack ps el-meu-blog
