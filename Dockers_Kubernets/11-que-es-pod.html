Què és un Pod?

La unitat mínima de treball a Kubernetes. Via imperativa vs declarativa. Cicle de vida i operativa diària.

Contenidor vs Pod

🫛 L'analogia de la beina de pèsols

Kubernetes mai executa contenidors directament. Sempre els envolta en un Pod.

Cas habitual

Una beida amb un sol pèsol: un Pod amb un sol contenidor (ex: un servidor web).

Casos avançats

Una beida amb diversos pèsols: un Pod amb un contenidor principal i un "sidecar" que comparteixen la mateixa IP i disc.

Regla d'or

Quan escales, no dius "crea més contenidors", sinó "crea més Pods". El Pod és la unitat mínima.

Via imperativa vs declarativa

⚡ Via Imperativa (ràpida)

Ordre directa. Útil per a proves ràpides.

sudo kubectl run el-meu-nginx --image=nginx
sudo kubectl get pods

📝 Via Declarativa (recomanada)

Fitxer YAML que declara l'estat desitjat. Per a producció real.

sudo kubectl apply -f nginx-pod.yaml
sudo kubectl get pods

📄 El nostre primer YAML — nginx-pod.yaml

apiVersion: v1
kind: Pod
metadata:
  name: nginx-declaratiu
  labels:
    app: web
spec:
  containers:
  - name: servidor-web
    image: nginx:latest
    ports:
    - containerPort: 80
sudo kubectl apply -f nginx-pod.yaml
Operativa diària

🔄 Les 3 comandes més usades

1

Llegir els Logs

sudo kubectl logs nginx-declaratiu
# Seguir en temps real:
sudo kubectl logs -f nginx-declaratiu
2

Entrar dins del contenidor

sudo kubectl exec -it nginx-declaratiu -- /bin/sh
# Un cop a dins:
ls
cat /etc/os-release
exit
3

Esborrar

sudo kubectl delete pod el-meu-nginx
sudo kubectl delete -f nginx-pod.yaml
