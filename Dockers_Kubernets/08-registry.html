Publicar a Docker Hub

🚀 El cicle Login → Build → Tag → Push

1

Autenticació

docker login
# Introdueix el teu usuari i contrasenya de Docker Hub
2

Construcció

docker build -t la-meva-app .
3

Etiquetatge (Tag)

Format: docker tag <imatge_local> <usuari>/<nom_imatge>:<versió>

docker tag la-meva-app guillemhs/hola-mon:v1.0
4

Pujada (Push)

docker push guillemhs/hola-mon:v1.0

Ara qualsevol persona del món pot fer docker run guillemhs/hola-mon:v1.0.

Bones pràctiques d'optimització

⚡ Els 4 Manaments de l'Optimització

1. No usaràs imatges gegants

Evita ubuntu:latest. Usa alpine o slim.

2. No executaràs com a Root

Per seguretat, crea un usuari dins del Dockerfile.

3. Usaràs .dockerignore

No copiis .git o node_modules. Ocupen espai i trenquen la caché.

4. Ordenaràs les capes

El que menys canvia (dependències) al principi. El codi al final. Màxima caché.

Comparativa visual

❌ Dockerfile dolent (~800MB)

FROM ubuntu:latest
RUN apt-get update
RUN apt-get install python3
COPY . /app
CMD ["python3", "/app/main.py"]

✅ Dockerfile optimitzat (~100MB)

FROM python:3.9-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
CMD ["python", "main.py"]
Tasca pràctica

🏋️ El Repte de l'Optimització

Un desenvolupador júnior t'ha passat un Dockerfile "desastre" que ocupa 900MB. Optimitza'l:

# Original (dolent)
FROM ubuntu:22.04
RUN apt-get update
RUN apt-get install -y python3 python3-pip
COPY . /app
WORKDIR /app
RUN pip3 install flask
CMD ["python3", "app.py"]

Entrega: El nou Dockerfile optimitzat + captura de docker images mostrant la diferència de mida.
