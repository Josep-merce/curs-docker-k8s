Networking Avançat
i Seguretat

Domina la comunicació entre contenidors. Tria el driver de xarxa adequat i segmenta aplicacions per seguretat (Frontend vs Backend vs DB).

Els 4 tipus de xarxes (Drivers)

🌉 Bridge — L'estàndard

Xarxa privada interna. Perfecta per a aplicacions web estàndard (WordPress + MySQL).

docker run --network bridge ...

🔥 Host — Rendiment màxim

El contenidor comparteix la IP i ports de la màquina física. Sense aïllament. Per a streaming o gaming.

docker run --network host ...
⚠️ Perill

Ocupa els ports de la màquina real directament.

🌍 Overlay — Clústers

Connecta contenidors en servidors físics diferents. Per a Docker Swarm i Kubernetes.

docker network create --driver overlay xarxa-cluster

🔇 None — Aïllament total

Només loopback. Cap connexió a l'exterior. Per a tasques de seguretat màxima.

docker run --network none ...
Gestió manual de xarxes

⚙️ Control total sobre IPs

# Crear xarxa amb IP fixa
docker network create \
  --driver bridge \
  --subnet=192.168.11.0/24 \
  --gateway=192.168.11.1 \
  xarxa-personalitzada

# Connectar/Desconnectar en calent
docker network connect xarxa-personalitzada el-meu-contenidor
docker network disconnect bridge el-meu-contenidor

# Inspeccionar
docker network inspect xarxa-personalitzada
Arquitectura segura de 3 nivells

🛡️ Segmentació de xarxes amb Docker Compose

La millor pràctica de seguretat: no posis tots els contenidors a la mateixa xarxa.

  • El món accedeix al Frontend.
  • El Frontend parla amb el Backend.
  • El Backend parla amb la DB.
  • 🚫 El Frontend NO pot parlar amb la DB directament.
version: "3.9"
services:
  frontend:
    image: nginx:latest
    ports:
      - "8080:80"
    networks:
      - front-tier

  backend:
    image: node:18-alpine
    networks:
      - front-tier
      - back-tier

  db:
    image: mysql:latest
    environment:
      MYSQL_ROOT_PASSWORD: secret
    networks:
      - back-tier

networks:
  front-tier:
  back-tier:
    internal: true  # Prohibeix sortida a internet
Troubleshooting de xarxes

🚑 Checklist de diagnòstic

1

Estan a la mateixa xarxa?

docker network inspect nom-de-la-xarxa
2

Resolució de noms (DNS)

docker exec -it contenidorA ping contenidorB
3

Ports exposats?

Recorda: entre contenidors NO cal exposar ports amb -p. El -p és NOMÉS per accedir des de fora.

4

Revisa els logs

docker logs contenidor-backend
