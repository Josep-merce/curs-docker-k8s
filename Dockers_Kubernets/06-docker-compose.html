Orquestració de Contenidors

Defineix i executa aplicacions multi-contenidor amb un sol fitxer YAML. Infraestructura com a codi: un sol comandament per aixecar tot l'entorn.

Conceptes clau

🐙 Què és Docker Compose?

Fins ara hem fet servir docker run per a cada servei. Docker Compose ens permet definir tota la infraestructura en un sol fitxer docker-compose.yml.

📄 Tot en un fitxer

Infraestructura com a codi (IaC). Tot el stack en un YAML versionable.

🚀 Un sol comandament

docker-compose up aixeca tot l'entorn complet.

🔗 Xarxes automàtiques

Els serveis es veuen entre ells pel seu nom (DNS intern automàtic).

⌨️ Comandes essencials

ComandaAcció
docker-compose upCrea i arrenca tots els serveis (afegeix -d per segon pla)
docker-compose downAtura i esborra contenidors i xarxes
docker-compose psLlista l'estat dels serveis de l'aplicació actual
docker-compose logs -fMostra els logs de tots els serveis unificats
docker-compose buildReconstrueix les imatges
docker-compose exec servei bashEntra dins d'un servei

🏆 WordPress complet amb Docker Compose

version: "3.8"

services:
  db:
    image: mysql:8.0
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: wordpress
      MYSQL_USER: wpuser
      MYSQL_PASSWORD: wppassword
    volumes:
      - db_data:/var/lib/mysql
    networks:
      - wp-network

  wordpress:
    image: wordpress:latest
    restart: always
    depends_on:
      - db
    ports:
      - "80:80"
    environment:
      WORDPRESS_DB_HOST: db:3306
      WORDPRESS_DB_USER: wpuser
      WORDPRESS_DB_PASSWORD: wppassword
      WORDPRESS_DB_NAME: wordpress
    volumes:
      - wordpress_data:/var/www/html
    networks:
      - wp-network

  db_data:
  wordpress_data:

  wp-network:

Escalabilitat i ordre d'arrencada

⚠️ El problema de l'ordre d'arrencada

Un problema comú: l'app intenta connectar-se a la base de dades abans que estigui llesta. Solució: usa depends_on i eines com wait-for-it per a esperes reals.

📈 Escalabilitat amb rèpliques

# Crear 3 contenidors del servei "web"
docker-compose up -d --scale web=3
⚠️ El problema dels ports

Si tens ports: "8080:80" al YAML, fallarà perquè només un contenidor pot ocupar el port 8080. Per escalar correctament necessites un load balancer (Nginx) davant.
