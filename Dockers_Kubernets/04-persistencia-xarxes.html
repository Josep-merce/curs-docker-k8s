docker run vs docker exec

La gran confusió resolta

ComandaQuè fa?Analogia
docker runCrea un NOU contenidor des de zeroConstrueix una casa nova i hi entra
docker execEntra en un contenidor JA EXISTENT (en marxa)Entra a una habitació d'una casa ja construïda
 
# Creem un contenidor
docker run -d --name el-meu-web nginx

# Entrem dins per modificar fitxers o depurar
docker exec -it el-meu-web bash

# Nota: si la imatge és molt bàsica (Alpine), usa sh
docker exec -it contenidor-alpine sh

📋 docker logs — Veure la sortida

# Veure logs
docker logs el-meu-web

# Seguir logs en temps real (com tail -f)
docker logs -f el-meu-web

# Últimes 50 línies
docker logs --tail 50 el-meu-web

# Amb timestamps
docker logs -t el-meu-web
Persistència de dades (Volums)

💾 Exemple pràctic: WordPress + MySQL

1

Crear els volums

docker volume create wordpress_data
docker volume create mysql_data
2

Base de Dades (MySQL)

docker run -d \
  --name wordpress_db \
  -e MYSQL_ROOT_PASSWORD=rootpassword \
  -e MYSQL_DATABASE=wordpress \
  -e MYSQL_USER=wordpressuser \
  -e MYSQL_PASSWORD=wordpresspassword \
  -v mysql_data:/var/lib/mysql \
  mysql:5.7
3

Aplicació (WordPress)

docker run -d \
  --name wordpress \
  --link wordpress_db:mysql \
  -e WORDPRESS_DB_HOST=wordpress_db:3306 \
  -e WORDPRESS_DB_USER=wordpressuser \
  -e WORDPRESS_DB_PASSWORD=wordpresspassword \
  -e WORDPRESS_DB_NAME=wordpress \
  -v wordpress_data:/var/www/html \
  -p 8080:80 \
  wordpress:latest
✅ Resultat

Accedeix a http://localhost:8080. Encara que esborris els contenidors, la web seguirà allà quan els tornis a aixecar gràcies als volums.

Networking i Ports

🔌 Exposició de Ports (-p)

La sintaxi és sempre: -p HOST:CONTENIDOR

# Bàsic
docker run -d -p 8080:80 nginx

# Múltiples ports
docker run -d -p 8080:80 -p 8443:443 httpd:latest

# Restringir a una IP específica
docker run -d --publish 192.168.1.100:8080:80 httpd:latest

Xarxes personalitzades (Bridge)

# Crear la xarxa
docker network create la-meva-xarxa

# Llançar el servidor a la xarxa
docker run -d --name servidor-web --network la-meva-xarxa httpd:latest

# Llançar un client per comprovar la connexió
docker run -it --network la-meva-xarxa alpine sh

# Dins del client alpine:
apk add curl
curl http://servidor-web
Tasca pràctica

🛠️ Entorn segur amb xarxa personalitzada

1

Crea la xarxa

docker network create xarxa-segura
2

Llança Apache amb Bind Mount al port 9090

docker run -d --name servidor-apache \
  --network xarxa-segura \
  -v $(pwd)/html:/usr/local/apache2/htdocs/ \
  -p 9090:80 \
  httpd

3

Prova la connexió des d'Alpine
