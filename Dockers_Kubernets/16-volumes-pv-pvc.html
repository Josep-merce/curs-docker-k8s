Volumes, PV i PVC
Persistència a Kubernetes

El sistema de fitxers d'un contenidor és efímer. Aprèn a treure les dades de dins del contenidor per garantir que sobrevisquin als reinicis i destruccions.

Stateless vs Stateful

🌊 Stateless (Sense estat)

Nginx, Apache, microserveis de càlcul. No guarden dades locals importants. Si el Pod mor, no passa res.

💾 Stateful (Amb estat)

MySQL, PostgreSQL, Redis. Necessiten persistència. Si el Pod mor i perdem el disc, és una catàstrofe.

Les tres capes de persistència
ComponentQuè és?Analogia
PersistentVolume (PV)Tros d'emmagatzematge real al clúster (disc local, AWS EBS, NFS)El disc dur real a la prestatgeria de la botiga
PersistentVolumeClaim (PVC)Petició d'emmagatzematge feta per un Pod ("necessito 5GB")El tiquet de compra o reserva del client
VolumeMountConfiguració dins del Pod per usar el PVCEl cable que connecta el disc a l'ordinador
Pràctica: Redis immortal

🛠️ Base de dades persistent amb Redis

1

Crear el PVC (redis-pvc.yaml)

apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: redis-dades-pvc
spec:
  accessModes:
  - ReadWriteOnce
  resources:
    requests:
      storage: 1Gi
sudo kubectl apply -f redis-pvc.yaml
2

Desplegar Redis (redis-pod.yaml)

apiVersion: v1
kind: Pod
metadata:
  name: redis-persistent
spec:
  containers:
  - name: redis
    image: redis:alpine
    volumeMounts:
    - name: emmagatzematge-redis
      mountPath: /data
  volumes:
  - name: emmagatzematge-redis
    persistentVolumeClaim:
      claimName: redis-dades-pvc
sudo kubectl apply -f redis-pod.yaml
3

La prova de foc

# Guardar una dada
sudo kubectl exec -it redis-persistent -- redis-cli set el_meu_nom "Goku"

# Esborrar el Pod!
sudo kubectl delete pod redis-persistent

# Recrear el Pod
sudo kubectl apply -f redis-pod.yaml

# Comprovar que la dada ha sobreviscut
sudo kubectl exec -it redis-persistent -- redis-cli get el_meu_nom
# Ha de retornar: "Goku"
StatefulSets (avançat)

📊 3 rèpliques amb 3 PVCs independents

apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: redis-statefulset
spec:
  serviceName: "redis"
  replicas: 3
  selector:
    matchLabels:
      app: redis
  template:
    metadata:
      labels:
        app: redis
    spec:
      containers:
      - name: redis
        image: redis:alpine
        volumeMounts:
        - name: emmagatzematge-redis
          mountPath: /data
  volumeClaimTemplates:
  - metadata:
      name: emmagatzematge-redis
    spec:
      accessModes: ["ReadWriteOnce"]
      resources:
        requests:
          storage: 1Gi
Tasca pràctica

🎯 Sobreviure al Caos — MariaDB

Desplega MariaDB amb un PVC de 2Gi, crea una base de dades prova_supervivencia, esborra el Pod i demostra que la base de dades ha sobreviscut en el Pod nou.

# Entrar a MariaDB
kubectl exec -it <nom-pod> -- mysql -u root -p

# Dins de MariaDB:
CREATE DATABASE prova_supervivencia;
SHOW DATABASES;
exit

# Destruir i recrear el Pod...
# Tornar a entrar i:
SHOW DATABASES;  # prova_supervivencia ha de seguir aquí!
