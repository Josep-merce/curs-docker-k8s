ReplicaSets
i Deployments

Els Pods solitaris moren i no tornen. Els ReplicaSets els ressusciten. Els Deployments gestionen versions, actualitzacions i rollbacks de forma elegant.

Deployment en pràctica

📝 Manifest wordpress-deployment.yaml

apiVersion: apps/v1
kind: Deployment
metadata:
  name: el-meu-wordpress
  labels:
    app: wordpress
spec:
  replicas: 3
  selector:
    matchLabels:
      app: wordpress
  template:
    metadata:
      labels:
        app: wordpress
    spec:
      containers:
      - name: wordpress
        image: wordpress:6.2-apache
        ports:
        - containerPort: 80
sudo kubectl apply -f wordpress-deployment.yaml
sudo kubectl get pods
# Hauries de veure 3 Pods creats automàticament!
💡 Prova-ho!

Esborra un Pod amb kubectl delete pod <nom> i torna a fer get pods ràpidament. El Deployment ja n'estarà creant un de nou!

Escalat

📈 Escalar amunt i avall

# Escalar a 5 rèpliques
sudo kubectl scale deployment el-meu-wordpress --replicas=5

# Verificar
sudo kubectl get pods

# Tornar a 2
sudo kubectl scale deployment el-meu-wordpress --replicas=2
Rolling Updates i Rollbacks

🔄 La superpotència dels Deployments

Rolling Update (actualització progressiva)

Kubernetes crea un Pod nou, espera que estigui llest, apaga un Pod antic, i repeteix. Zero downtime!

# Actualitzar de v6.2 a v6.3
sudo kubectl set image deployment/el-meu-wordpress wordpress=wordpress:6.3-apache

# Veure l'estat en temps real
sudo kubectl rollout status deployment/el-meu-wordpress

Rollback (desfer l'últim canvi)

# La v6.3 té un error crític? No pateixis!
sudo kubectl rollout undo deployment/el-meu-wordpress

# Veure l'historial
sudo kubectl rollout history deployment/el-meu-wordpress
Tasca pràctica

🏋️ Gestió de Deployments

Crea tasca10.yaml amb un Deployment d'Apache amb 2 rèpliques, aplica'l, escala a 5 i captura el resultat.

apiVersion: apps/v1
kind: Deployment
metadata:
  name: apache-deployment
spec:
  replicas: 2
  selector:
    matchLabels:
      app: apache
  template:
    metadata:
      labels:
        app: apache
    spec:
      containers:
      - name: apache
        image: httpd:alpine
        ports:
        - containerPort: 80
sudo kubectl apply -f tasca10.yaml
sudo kubectl scale deployment apache-deployment --replicas=5
sudo kubectl get pods
