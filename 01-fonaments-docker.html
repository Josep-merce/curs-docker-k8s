Fonaments de Docker
Creació i Gestió de Contenidors

Aprèn a interactuar directament amb el Docker Engine, dominant el cicle de vida essencial d'un contenidor: des del seu naixement fins a la seva aturada segura.

Objectius d'aprenentatge

🚀 Desplegament de Contenidors

  • Iniciar contenidors amb un sol comandament
  • Diferència entre execució en primer pla i detached mode (-d)
  • Mapeig de ports (-p), noms personalitzats (--name) i neteja automàtica (--rm)

👁️ Monitorització i Visibilitat

  • Llistar contenidors actius i inactius (docker ps)
  • Ús de filtres avançats
  • Interpretació de l'estat (Up, Exited) i els IDs

⚙️ Gestió del Cicle de Vida

  • Aturada segura amb Graceful Shutdown
  • Diferència entre SIGTERM (stop) i SIGKILL (kill)
  • Gestió de temps d'espera
Comandes essencials

docker run

Crea i inicia un contenidor nou des d'una imatge. És la comanda central de Docker.

Sintaxi bàsica

docker run [opcions] IMATGE [comanda]

Opcions principals

OpcióDescripcióExemple
-dMode desacoblat (segon pla)docker run -d nginx
-p HOST:CONTMapeja portsdocker run -p 8080:80 nginx
--name NOMAssigna un nomdocker run --name web nginx
--rmEsborra en aturar-sedocker run --rm ubuntu echo "hola"
-itTerminal interactiudocker run -it ubuntu bash
-e VAR=VALORVariable d'entorndocker run -e MYSQL_ROOT_PASSWORD=secret mysql

Exemples pràctics

# Hello World efímer
docker run --rm hello-world

# Nginx accessible al port 80 del host
docker run -d --name servidor-web -p 80:80 nginx

# Terminal interactiu a Ubuntu
docker run -it --rm ubuntu bash

docker ps — Llistar contenidors

# Contenidors actius
docker ps

# Tots (incloent aturats)
docker ps -a

# Filtrar per nom
docker ps --filter "name=web"

# Format personalitzat
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"

La sortida mostra: CONTAINER ID, IMAGE, COMMAND, CREATED, STATUS, PORTS, NAMES.

Gestió del cicle de vida

ComandaSenyalComportamentQuan usar-la
docker stopSIGTERM → SIGKILL (10s)Aturada elegant: l'aplicació pot netejar recursosSempre que sigui possible
docker killSIGKILL (immediat)Aturada forçada: el procés mor immediatamentQuan stop no respon
docker rm—Esborra el contenidor aturatNeteja
docker rm -f—Atura i esborraNeteja ràpida
 
# Aturar de manera elegant
docker stop servidor-web

# Aturar amb temps d'espera personalitzat (30 seg)
docker stop -t 30 servidor-web

# Forçar l'aturada
docker kill servidor-web

# Esborrar contenidor aturat
docker rm servidor-web

# Esborrar tots els contenidors aturats
docker container prune
Tasca pràctica

🛠️ Exercici de la sessió

1

Hello World efímer

Executa el contenidor clàssic i observa la sortida.

docker run --rm hello-world
2

Servidor web Nginx

Desplega un Nginx accessible des del navegador a localhost:80.

docker run -d --name servidor-web -p 80:80 nginx
3

Neteja

Atura i elimina tots els contenidors que has creat.

docker stop servidor-web
docker rm servidor-web
💡 Consell

Fes servir sempre --name per posar noms descriptius als contenidors. Serà molt més fàcil identificar-los amb docker ps que recordar els IDs hexadecimals.
