ConfigMaps
i Secrets

Separa el codi de la configuració. Injecta variables d'entorn, fitxers de configuració i contrasenyes sense tocar ni una línia de codi.

La regla d'or: Codi vs Configuració

🧩 Twelve-Factor App

📦 El codi — Immutable

La imatge del contenidor no hauria de canviar mai. Igual al local que a producció.

⚙️ La configuració — Variable

URLs, modes de depuració, credencials. Canvia entre entorns. S'injecta en temps d'execució.

ConfigMaps vs Secrets
CaracterísticaConfigMapSecret
Tipus de dadesPúbliques / No sensiblesSensibles (contrasenyes, tokens, certificats)
EmmagatzematgeText plaCodificat en Base64
Cas d'úsVariables d'entorn, fitxers .conf, URLsCredencials de DB, claus SSH, TLS
⚠️ Important sobre Base64

Base64 és una codificació, no una encriptació forta. Serveix per evitar que les contrasenyes es llegeixin a simple vista, però no és seguretat real. Kubernetes ofereix mecanismes addicionals per encriptar Secrets al disc.

Pràctica: Injectar configuració

🛠️ Crear ConfigMap i Secret

1

Crear de forma imperativa

# ConfigMap
sudo kubectl create configmap configuracio-web \
  --from-literal=MISSATGE="Hola des de Kubernetes!"

# Secret (Kubernetes codifica en Base64 automàticament)
sudo kubectl create secret generic credencials-db \
  --from-literal=PASSWORD="SuperSecreta123"
2

Crear app-configurada.yaml

apiVersion: v1
kind: Pod
metadata:
  name: pod-amb-configuracio
spec:
  containers:
  - name: el-meu-contenidor
    image: alpine
    command: ["/bin/sh", "-c", "echo El missatge: $MISSATGE_APP i la clau: $CLAU_DB; sleep 3600"]
    env:
    - name: MISSATGE_APP
      valueFrom:
        configMapKeyRef:
          name: configuracio-web
          key: MISSATGE
    - name: CLAU_DB
      valueFrom:
        secretKeyRef:
          name: credencials-db
          key: PASSWORD
3

Aplicar i comprovar els logs

sudo kubectl apply -f app-configurada.yaml
sudo kubectl logs pod-amb-configuracio
Tasca pràctica

🎯 ConfigMaps i Secrets en YAML

Codifica el teu secret en Base64:

echo -n "SuperSecreta2024" | base64

Crea tres fitxers YAML: configmap-botiga.yaml (IDIOMA: "ca_ES", TEMA: "fosc"), secret-botiga.yaml (DB_PASS en Base64) i pod-botiga.yaml que imprimeixi les tres variables.
