Troubleshooting
Resolució de Problemes

Les coses fallen. Saber diagnosticar i arreglar errors és el que separa un principiant d'un veritable expert. Converteix-te en un detectiu del clúster.

Estats d'error més comuns

⏳ Pending

El Pod ha estat acceptat però no s'ha pogut assignar a cap node.

Causa típica: Falta de recursos (CPU/RAM) o PVC que no es pot satisfer.

🖼️ ImagePullBackOff

El clúster ha intentat descarregar la imatge però ha fracassat.

Causa típica: Nom de la imatge incorrecte, etiqueta inexistent, o repositori privat sense credencials.

🔁 CrashLoopBackOff

El contenidor arrenca però mor immediatament. Kubernetes ho reintenta en bucle.

Causa típica: Error de codi, variable d'entorn obligatòria faltant, comanda d'inici incorrecta.

🛠️ Comandes detallades

# Veure tots els pods (incloent els que fallen)
kubectl get pods -A

# Radiografia completa d'un Pod
kubectl describe pod nom-del-pod

# Llegir logs
kubectl logs nom-del-pod
kubectl logs -f nom-del-pod  # En temps real
kubectl logs nom-del-pod --previous  # De la instància anterior (si ha mort)

# Events ordenats per temps
kubectl get events --sort-by='.metadata.creationTimestamp'

# Editar un recurs en viu
kubectl edit deployment nom-del-deployment

🎮 Metodologia de diagnòstic

1

Investigar

kubectl get pods
# Identifica quins recursos estan fallant i quin és el seu estat
2

Diagnosticar

kubectl describe pod nom-del-pod
kubectl logs nom-del-pod
# Llegeix els Events i els missatges d'error
3

Reparar

kubectl edit deployment nom-del-deployment
# Corregeix l'error tipogràfic, afegeix la variable faltant, etc.
4

Verificar

kubectl get pods
# El Pod ha de passar a l'estat Running i mantenir-s'hi
💡 Consell d'expert

Respireu fons quan les coses fallen! Kubernetes és un sistema increïblement xerraire si sabem on escoltar. El procés lògic és: observar → events → describe → logs. No esborreu res fins a entendre el problema.
