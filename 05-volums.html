Volums
Persistència de Dades

Els contenidors són efímers per defecte. Aprèn a usar Volumes, Bind Mounts i tmpfs per garantir que les teves dades sobrevisquin.

El problema de l'efimeritat

🏨 L'analogia de l'hotel

Un contenidor és com una habitació d'hotel:

  • Pots portar les teves coses i moure mobles mentre hi ets.
  • Quan fas el "check-out" (elimines el contenidor), l'habitació es neteja completament per al següent hoste.

Per guardar dades de manera permanent (com una base de dades o logs), necessitem un mecanisme extern: els Volums.

🔧 Volumes gestionats per Docker

1

Crear i inspeccionar

docker volume create les_meves_dades
docker volume ls
docker volume inspect les_meves_dades
2

Connectar a un contenidor

docker run -d --name contenidor_1 -v les_meves_dades:/data alpine tail -f /dev/null
docker exec contenidor_1 sh -c "echo 'Hola des del contenidor 1!' > /data/missatge.txt"
docker exec contenidor_1 cat /data/missatge.txt
3

Prova de foc: esborra el contenidor i recrea'l

docker rm -f contenidor_1
docker run -d --name contenidor_2 -v les_meves_dades:/data alpine tail -f /dev/null
docker exec contenidor_2 cat /data/missatge.txt
# Hauries de veure: "Hola des del contenidor 1!"
4

Neteja

docker rm -f contenidor_2
docker volume rm les_meves_dades

📁 Bind Mounts — Edició en viu

Perfecte per al desenvolupament: edita al teu ordinador i els canvis es reflecteixen al contenidor a l'instant.

# Muntar el directori actual a Nginx
docker run -d \
  --name dev-web \
  -v $(pwd)/web_project:/usr/share/nginx/html \
  -p 8080:80 \
  nginx
💡 Consell

Usa sempre rutes absolutes o $(pwd) per als bind mounts. Les rutes relatives no funcionen amb Docker.

Bones pràctiques

✅ Consells d'expert

  • Noms explícits: Usa noms descriptius (ex: postgres_data, web_logs). Si no, Docker genera hashes aleatoris impossibles de recordar.
  • Còpies de seguretat: Els volums són persistents però no indestructibles. Fes backups del directori /var/lib/docker/volumes.
  • Drivers: Docker permet drivers per guardar dades al núvol (AWS S3, Azure) o en xarxa (NFS).
